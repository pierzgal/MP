GIT – RStudio Tutorial 1

Tutorial: Deploying Quarto Project’s Docs Folder to GitHub

Introduction to Git Commands

Before we dive into the specific project, let’s cover the main Git commands you’ll be using:

  1. git init: Initializes a new Git repository in the current directory.

  2. git clone [url]: Creates a copy of a remote repository on your local machine.

  3. git add [file]: Stages changes for commit. Use git add . to stage all changes.

  4. git commit -m "[message]": Creates a new commit with the staged changes and a descriptive message.

  5. git push [remote] [branch]: Uploads local commits to a remote repository.

  6. git pull [remote] [branch]: Fetches changes from a remote repository and merges them into your current branch.

  7. git status: Shows the status of changes as untracked, modified, or staged.

  8. git branch: Lists all local branches in the repository.

  9. git checkout -b [branch-name]: Creates a new branch and switches to it.

  10. git remote add [name] [url]: Adds a new remote repository.

Now, let’s apply these commands to our specific project.

Deploying Quarto Project’s Docs Folder to GitHub

Prerequisites

  • An existing Quarto website project on your local machine
  • An empty GitHub repository
  • RStudio installed on your computer
  • Git installed on your computer

Step 1: Configure Quarto Project

Ensure your Quarto project is set up to render to the docs folder:

  1. Open your Quarto project in RStudio.
  2. In your _quarto.yml file, add or modify the following:
project:
  output-dir: docs

Step 2: Initialize Git in Your Local Project

Git command:

git init

RStudio: 1. Go to Tools > Version Control > Project Setup 2. Choose “Git” as the version control system 3. Allow RStudio to restart

Step 3: Create .gitignore File

To ensure we only push the docs folder, create a .gitignore file in your project root:

Git command:

echo "*
!docs/
!.gitignore" > .gitignore

RStudio: 1. File > New File > Text File 2. Add the following content: * !docs/ !.gitignore 3. Save as .gitignore in the project root

This .gitignore file tells Git to ignore everything except the docs folder and the .gitignore file itself.

Step 4: Render Quarto Project

In RStudio: 1. Click the “Render” button, or 2. In the Console, run: r quarto render

This will create or update the docs folder with your rendered website.

Step 5: Stage Changes

Git command:

git add docs/ .gitignore

RStudio: In the Git pane, check the boxes next to the docs folder and .gitignore file.

Step 6: Commit Changes

Git command:

git commit -m "Add rendered website files and .gitignore"

RStudio: 1. Click the “Commit” button in the Git pane 2. Enter a commit message 3. Click “Commit”

Step 7: Connect to Remote Repository

Git command:

git remote add origin https://github.com/yourusername/your-repo-name.git

RStudio: In the Console, run:

system('git remote add origin https://github.com/yourusername/your-repo-name.git')

Replace the URL with your actual GitHub repository URL.

Step 8: Create and Switch to Main Branch

Git command:

git checkout -b main

RStudio: In the Console, run:

system("git checkout -b main")

Step 9: Push to GitHub

Git command:

git push -u origin main

RStudio: 1. Click the “Push” button in the Git pane 2. For the first push, in the Console, run: r system('git push -u origin main')

Step 10: Configure GitHub Pages

  1. Go to your GitHub repository in a web browser
  2. Click on “Settings”
  3. In the left sidebar, click on “Pages”
  4. Under “Source”, select “Deploy from a branch”
  5. Under “Branch”, select “main” and “/docs” folder
  6. Click “Save”

Your Quarto website should now be live at https://yourusername.github.io/your-repo-name/.

GitHub Authentication

When working with GitHub from RStudio, you’ll need to authenticate to push changes to your remote repository. GitHub no longer supports password authentication for Git operations, so you’ll need to use a personal access token or SSH key. Here’s how to set up authentication:

Option 1: Personal Access Token (Recommended for beginners)

  1. Generate a Personal Access Token (PAT) on GitHub:
    • Go to GitHub.com and log in
    • Click on your profile picture > Settings > Developer settings > Personal access tokens
    • Click “Generate new token”
    • Give your token a descriptive name
    • Select the necessary scopes (at minimum, select “repo”)
    • Click “Generate token”
    • Copy the token immediately (you won’t be able to see it again)
  2. Store the PAT in RStudio:

    • In RStudio, install the usethis package if you haven’t already:

      install.packages("usethis")

    • Run the following command to open your Git credentials file:

      usethis::edit_git_config()

    • Add the following lines, replacing YOUR_TOKEN with your actual token:

      [github]
   token = YOUR_TOKEN

    • Save and close the file

  3. Use the token:
    • RStudio will now use this token automatically when pushing to GitHub
    • If prompted for a password, use the token instead

Option 2: SSH Key

  1. Generate an SSH key pair:

    • In RStudio’s Terminal, run:

      ssh-keygen -t ed25519 -C "your_email@example.com"

    • Press Enter to accept the default file location

    • Enter a secure passphrase (or press Enter for no passphrase)

  2. Add the SSH key to the ssh-agent:

    • Start the ssh-agent:

      eval "$(ssh-agent -s)"

    • Add your SSH private key to the ssh-agent:

      ssh-add ~/.ssh/id_ed25519
  3. Add the public key to your GitHub account:

    • Copy the public key to your clipboard:

      cat ~/.ssh/id_ed25519.pub

    • Go to GitHub.com and log in

    • Click on your profile picture > Settings > SSH and GPG keys

    • Click “New SSH key”

    • Paste your key into the “Key” field

    • Give your key a descriptive title

    • Click “Add SSH key”

  4. Use SSH URL for your repository:

    • When adding the remote repository, use the SSH URL instead of HTTPS:

      git remote add origin git@github.com:yourusername/your-repo-name.git

Troubleshooting Authentication Issues

If you encounter authentication issues:

  1. Verify your credentials:

    • For PAT: Check if the token is still valid on GitHub
    • For SSH: Ensure your SSH key is added to your GitHub account

  2. Reset credentials:

    • On Windows: Search for “Credential Manager” and remove any GitHub entries
    • On macOS: Use Keychain Access to remove any GitHub entries

  3. Use the git credential helper:

    git config --global credential.helper cache

    This will cache your credentials for a short time, reducing the number of times you need to enter them.

  4. If using HTTPS, ensure you’re not being prompted for a username. Your username should be your GitHub username or the email associated with your GitHub account.

By following these authentication steps, you should be able to securely push your Quarto project to GitHub from RStudio. Remember to keep your PAT or SSH private key secure and never share them publicly.

Updating Your Website

For future updates:

  1. Make changes to your Quarto project
  2. Render the project (Step 4)
  3. Stage changes (Step 5)
  4. Commit changes (Step 6)
  5. Push changes (Step 9)

GitHub will automatically update your published site after you push changes.

Remember to pull any changes if you’re working on multiple machines:

Git command:

git pull origin main

RStudio: Click the “Pull” button in the Git pane

This revised tutorial focuses on pushing only the docs folder to your GitHub repository and includes an introduction to the main Git commands. The .gitignore file ensures that only the necessary files are tracked and pushed to GitHub.
